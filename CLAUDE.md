# njfdCRH1A.github.io 项目规范指南

## 1. 编辑纪律

### 1.1 CSS 大段改动直接 Write，不用 Edit

CSS 选择器块由 `{` 到 `}` 配对，逐块 Edit 容易漏括号或错位。以下情况直接 **Write 整个文件**：

- 改动涉及 ≥3 个 CSS 规则块
- 涉及 CSS 变量（`:root` 块）的增删
- 涉及 `@media` 查询块的修改
- 改动 > 单文件 20%

小改动（单条属性、单个选择器）才用 Edit。

### 1.2 HTML 结构改动先列 plan 再执行

修改 HTML 前，先用 Read 确认各目标区域的行号和唯一上下文，列出清单：

```
Block A: header/nav 导航菜单 (行 11-28) → 确认 → Edit
Block B: main 内容区卡片 (行 35-50) → 确认 → Edit
Block C: footer 页脚 (行 111-113) → 确认 → Edit
```

- 任一步 Edit 失败 → 停，不准跳
- 所有失败必须回到 Read → 确认 → Edit 循环
- 导航菜单在 6 个 HTML 文件中重复出现，改动导航时**必须同步更新所有文件**

### 1.3 不改无关

- 不改没坏的东西
- 不"改进"相邻未要求的代码/注释/格式/类名
- 不擅自修复相邻的 CSS 属性顺序或空白行
- 发现因本次改动变成孤儿的 HTML 元素或 CSS 规则 → 可删，在改动汇报中说明
- 原本就死的资源文件（如图片、字体未引用）→ 先告知用户，不擅自删

### 1.4 改完自验

每改完一个文件，立即核对：

1. HTML：每个开始标签是否有对应的闭合标签（特别检查 `<div>`、`<section>`、`<ul>`）
2. CSS：每个 `{` 是否有配对的 `}`，缩进是否与上下文一致
3. CSS：新增/删除的属性是否在正确的规则块内
4. 浏览器打开相关页面，DevTools Console 确认无红色报错

## 3. 项目技术约束

| 约束 | 说明 |
|------|------|
| 零依赖 | 不引入任何 npm 包、CDN 链接、第三方 JS/CSS 库 |
| 纯静态 | 无需构建工具，浏览器直接打开 `.html` 文件 |
| 原生 CSS3 | CSS 变量 + Flexbox + Grid，无预处理器 |
| 无 JS 框架 | 主站页面为纯 HTML+CSS，无 JavaScript 逻辑（子项目除外） |
| 中文优先 | 页面内容为中文，代码注释可用中英混合 |

## 5. CSS 编写规范

### 5.1 分区注释

每个 CSS 分区用醒目的分隔注释：

```css
/* ========================
   Header & Navigation
   ======================== */
```

### 5.2 变量优先

颜色、阴影、圆角等设计令牌统一在 `:root` 中定义，不在具体规则中硬编码数值：

```css
:root {
  --primary: #1a1a2e;
  --accent: #e94560;
  --bg: #f8f9fa;
  --radius: 16px;
  --shadow: 0 4px 20px rgba(0,0,0,0.08);
  --transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

### 5.3 响应式

所有新样式需考虑移动端适配，在 `@media (max-width: 768px)` 中补充对应规则。

## 6. 验收方式

由于项目无测试框架，改动验收方式为：

1. 浏览器打开受影响的 `.html` 文件
2. 检查视觉一致性（与改动前对比）
3. 缩放浏览器窗口验证响应式布局不崩
4. 打开 DevTools Console 确认无红色报错
5. 检查所有导航链接是否正常跳转
